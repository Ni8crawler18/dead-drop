Architecture

as dead drops

01

Typed Witness Pattern

DeadDropAuth registered on Storage Units via authorize_extension. Only our contract can move items.

02

Event-Based Key Reveal

IntelPurchased event emitted on purchase containing the AES key. Only the buyer's transaction includes it.

03

Client-Side Decryption

dApp extracts key from event, decrypts ciphertext using Web Crypto API. Secret displayed instantly. No server involved.

Security Model

Trustless by design.

Encryption

AES-256-GCM — intel encrypted in the browser before going on-chain. Keys never stored in plaintext. Revealed only in purchase events.

Extension Auth

Typed witness pattern — DeadDropAuth restricts storage unit access. Only our contract can deposit/withdraw.

Self-Purchase Block

Smart contract prevents buying your own listings. Abort code checked on-chain. No gaming the system.

Rate-Limited API

Onboarding endpoint: 1 request/min per wallet. CORS restricted to dApp domain. Error messages truncated to prevent info leaks.
